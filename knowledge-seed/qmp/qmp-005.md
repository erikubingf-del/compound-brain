# QMP-005 — Project Intelligence Bootstrap

## Q — Question
How do I get up to speed on an unfamiliar codebase and make a useful contribution in the
first session?

## M — Model
**Read the constitution, then the evidence trail**. Every project has a constitution
(CLAUDE.md, README, or equivalent) that describes intent, and an evidence trail (git log,
daily notes, decision log) that describes reality. The gap between them is where the most
important current work lives.

Resist the urge to read code first. Read intent first, then map intent to code.

## P — Process

### 1. Read the constitution (5 min)
- `CLAUDE.md` or `README.md` — what is this project and why does it exist?
- `.brain/memory/project_context.md` — current state, risks, next actions
- `.brain/memory/feedback_rules.md` — how does the project owner like to work?

### 2. Read the evidence trail (5 min)
```bash
# What changed recently?
git log --oneline -20

# What's in flight?
git status && git diff --stat

# What was decided?
cat .brain/knowledge/decisions/log.md | tail -50
```

### 3. Read the latest intelligence brief (2 min)
- `.brain/knowledge/daily/intelligence_brief_latest.md`
- Contains the AI's last analysis of project state, risks, and recommended next action

### 4. Map intent to code (10 min)
- Find the entry points: `main.py`, `index.ts`, `app.py`, or equivalent
- Trace one key flow end-to-end: input → processing → output
- Note: what matches the constitution? What doesn't?

### 5. Pick the highest-EV action
Run the probability engine if available:
```bash
python3 ~/.claude/scripts/probability_engine.py --project-dir . --output text
```
Or manually rank: what is the highest-impact, highest-probability-of-success, most urgent thing to do?

### 6. Record what you learned
Before ending the session, update `.brain/memory/project_context.md` with anything
that was missing or wrong when you started.

## Applied To
- Any project with a .brain/ scaffold
- Works without a brain — just read README + git log instead

## Pitfalls
- Starting with the code before reading the constitution — leads to wrong assumptions baked in
- Treating the README as current reality — it is often aspirational or stale
- Skipping the evidence trail — misses what was decided and why
- Not updating project_context.md after the session — the next agent starts cold again
