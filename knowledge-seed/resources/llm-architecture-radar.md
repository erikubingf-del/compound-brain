# LLM Architecture Radar

## Purpose
Track high-signal agent, memory, orchestration, and verification patterns from
recent GitHub projects and convert only the durable ones into improvements for
the shared orchestrator.

## Selection Heuristics
- prefer architecture patterns over novelty
- prefer fixed-evaluator and verification-heavy systems
- prefer ideas that improve memory, control-plane integrity, or observability
- reject patterns that create parallel brains or unbounded autonomy

## Output
- ranked architecture findings
- proposed shared-runtime upgrades
- candidate promotions for global skills, QMP, or decisions
