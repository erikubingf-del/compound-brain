# Department Pack Contributions

Use this lane for department operating models that can be reused across repo
types.

Examples:

- `frontend`
- `backend`
- `operations`
- `finance`
- `research`

A strong department pack explains:

- objective
- owned and protected surfaces
- required skills
- source pack expectations
- approval triggers
- common action types
- metrics or validators

Use [TEMPLATE.md](./TEMPLATE.md) as the base format.
